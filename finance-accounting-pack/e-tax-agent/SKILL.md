> **e-Tax電子申告連携エージェント** -- 国税庁e-Taxシステムと連携し、確定申告書の作成から電子送信、受信通知確認、納税証明書取得、予定納税管理までをClaude Codeの自然言語インターフェースで一元操作。マイナンバーカードによる電子署名にも対応し、税務手続きのデジタル化を完全にサポートします。

---

## Overview

E-Tax Agentは、国税庁が提供するe-Tax（国税電子申告・納税システム）のAPIと連携し、個人事業主・法人の税務手続きをClaude Code上で完結させるスキルです。確定申告書（所得税、消費税、法人税）の作成、電子署名、送信、受信通知の確認、各種証明書の取得、予定納税の管理、更正の請求まで、税務に関するあらゆる操作を自然言語コマンドで実行できます。

e-Tax APIの仕様に準拠したXTX形式のデータ生成、マイナポータルAPIを経由したマイナンバーカード連携、JPKI（公的個人認証サービス）による電子署名を統合しています。

## System Prompt / Behavioral Guidelines

このスキルを使用する際、エージェントは以下のガイドラインに従います。

1. すべての税務データは国税庁のe-Tax仕様（XTX形式）に準拠して生成し、送信前に必ずバリデーションを実行する
2. マイナンバー（個人番号）は暗号化して保持し、ログファイルやコンソール出力には絶対に平文表示しない（先頭4桁のみマスク表示を許容）
3. 電子署名はJPKI（公的個人認証サービス）の仕様に従い、SHA-256によるハッシュ値を使用する
4. 申告書データの送信前に必ず金額の整合性チェック（収入-経費=所得、税額計算の検算）を実施する
5. e-Tax APIへの通信はすべてTLS 1.2以上で暗号化し、証明書の有効性を検証する
6. 申告期限（所得税: 3月15日、消費税: 3月31日、法人税: 事業年度終了後2ヶ月）を常に表示し、期限間近の場合は警告を発する
7. 税額計算は最新の税率テーブル（所得税率、住民税率、事業税率、消費税率）を使用し、年度ごとの改正に対応する
8. 青色申告特別控除（65万円/55万円/10万円）の適用要件を自動判定し、最適な控除額を提案する
9. 予定納税の通知時期（6月、11月）と納期限を管理し、事前にリマインド通知を行う
10. 更正の請求は法定申告期限から5年以内であることを確認し、期限外の場合はエラーを返す
11. 操作ログは税務調査対応として7年間保持し、タイムスタンプ付きで全操作を記録する
12. 消費税の計算は標準税率（10%）と軽減税率（8%）を正確に区分し、仕入税額控除の計算を行う
13. 住民税・事業税の申告データは確定申告データから自動生成し、地方税ポータルへの連携をサポートする
14. 電子納税（ダイレクト納付、インターネットバンキング、クレジットカード）の手続き案内を提供する
15. API障害時はe-Taxの稼働状況ページを確認し、メンテナンス情報を表示する
16. 確定申告書等作成コーナーとの互換性を維持し、途中データのインポート/エクスポートに対応する
17. すべての金額表示は日本円（円単位、3桁カンマ区切り）で統一し、端数処理は国税通則法に従う

## Environment Variables

| 変数名 | 必須 | 説明 | 取得方法 |
|--------|------|------|----------|
| `ETAX_USER_ID` | はい | e-Tax利用者識別番号（16桁） | e-Tax開始届出書の提出後に発行される利用者識別番号 |
| `ETAX_PASSWORD` | はい | e-Taxログインパスワード | e-Tax初期設定時に自身で設定したパスワード |
| `ETAX_CERT_PATH` | はい | 電子証明書ファイルのパス（.pfx/.p12形式） | マイナンバーカードから書き出した電子証明書ファイル |
| `MYNUMBER_CARD_PIN` | はい | マイナンバーカードの署名用パスワード（6～16桁英数字） | マイナンバーカード交付時に設定した署名用パスワード |
| `ETAX_TAX_OFFICE_CODE` | いいえ | 所轄税務署コード（5桁） | 国税庁サイトの税務署検索ページで確認 |
| `ETAX_FISCAL_YEAR` | いいえ | 対象事業年度（デフォルト: 前年） | YYYY形式（例: 2025） |
| `ETAX_BLUE_RETURN` | いいえ | 青色申告フラグ（true/false） | デフォルト: false |
| `ETAX_LOG_RETENTION_YEARS` | いいえ | ログ保持年数 | デフォルト: 7（税務調査対応） |

## Workflow Diagram


## FAQ

**Q1: e-Taxの利用者識別番号はどうやって取得しますか？**
A1: e-Tax公式サイト（https://www.e-tax.nta.go.jp/）から「開始届出書」をオンラインで提出すると、即時に16桁の利用者識別番号が発行されます。マイナンバーカードがあれば、マイナポータル経由でも取得可能です。

**Q2: マイナンバーカードなしでも電子申告できますか？**
A2: ID・パスワード方式を利用すれば、マイナンバーカードなしでも電子申告が可能です。ただし、事前に税務署での本人確認が必要です。`ETAX_AUTH_METHOD=id-password` を設定してください。

**Q3: 青色申告の65万円控除を受けるための要件は？**
A3: 複式簿記による記帳、貸借対照表・損益計算書の添付、e-Taxによる電子申告（または電子帳簿保存）が必要です。本スキルのe-Tax送信機能を使用すれば電子申告要件を満たせます。

**Q4: 消費税のインボイス制度にも対応していますか？**
A4: 消費税の確定申告書作成に対応しています。インボイス制度（適格請求書等保存方式）に関する詳細な機能は、別途japan-invoiceスキルをご利用ください。

**Q5: 過去の申告を修正したい場合は？**
A5: `etax amendment create` コマンドで更正の請求書（税額が減る場合）または修正申告書（税額が増える場合）を作成できます。法定申告期限から5年以内が対象です。

**Q6: 予定納税の減額申請はできますか？**
A6: はい。`etax payment reduce-request` コマンドで予定納税の減額申請書を作成・送信できます。事業の廃止・休止、業績不振等の理由が必要です。

**Q7: 住民税の申告も連動しますか？**
A7: 確定申告データから住民税に関する事項が自動反映されます。確定申告を行えば原則として住民税の申告は不要です。特別徴収/普通徴収の選択は `etax file create --residence-tax ordinary` で設定できます。

**Q8: データのバックアップはどうなっていますか？**
A8: すべての申告データは `~/.etax-agent/returns/` にXTX形式で保存されます。また、送信済みデータは `~/.etax-agent/submitted/` に自動バックアップされます。7年間の保持が税務調査対応として推奨されます。

**Q9: 法人税の申告にも対応していますか？**
A9: 法人税確定申告書（別表一～十六）の作成に対応しています。`etax file create --type corporate-tax --fiscal-end 2026-03-31` で法人税申告書を作成できます。

**Q10: e-Tax APIの利用時間に制限はありますか？**
A10: 通常期は火曜日～金曜日の24時間、土日月曜日はメンテナンスのため利用不可です。ただし、確定申告期間（1月上旬～3月中旬）は全日24時間利用可能です。最新の稼働状況は `etax status server` で確認できます。

## Advanced Features

### 会計ソフト連携

E-Tax Agentは主要な会計ソフトからのデータインポートに対応しています。freee会計、マネーフォワードクラウド会計、やよいの青色申告からCSVまたはAPI経由で仕訳データを取得し、確定申告書に必要な勘定科目の集計を自動で行います。

インポート対応フォーマットは以下の通りです。freee会計のエクスポートCSV形式、マネーフォワードクラウドの仕訳帳CSV形式、やよいの青色申告の仕訳データCSV形式、および汎用の勘定科目別集計CSV形式です。インポート時にはデータの整合性チェックが自動的に実行され、貸借不一致や不正な勘定科目コードなどの問題がある場合は修正案を提示します。

### 減価償却資産の自動計算

固定資産の減価償却計算を自動化します。定額法、定率法、生産高比例法に対応し、耐用年数省令に基づいた法定耐用年数テーブルを内蔵しています。取得価額、取得日、資産区分を入力するだけで、当期の償却費を自動計算し、確定申告書の減価償却明細に反映します。

中小企業者の少額減価償却資産の特例（取得価額30万円未満、年間合計300万円まで即時償却可能）にも対応しており、適用要件を自動判定して最適な償却方法を提案します。一括償却資産（取得価額20万円未満）の3年均等償却計算にも対応しています。

### 所得控除の自動適用

確定申告における各種所得控除の適用判定を自動化します。対応する所得控除は以下の通りです。基礎控除（合計所得金額に応じた段階的控除）、配偶者控除・配偶者特別控除（配偶者の所得に応じた自動計算）、扶養控除（特定扶養親族、老人扶養親族等の区分判定）、社会保険料控除、小規模企業共済等掛金控除（iDeCo含む）、生命保険料控除（一般・介護医療・個人年金の区分計算）、地震保険料控除、医療費控除（通常の医療費控除とセルフメディケーション税制の有利判定）、寄附金控除（ふるさと納税の控除上限額計算含む）。

ふるさと納税の控除上限額シミュレーション機能も搭載しており、`etax simulate furusato --income 8000000` で年間の控除上限額を即座に計算できます。ワンストップ特例との比較も自動で行い、確定申告が有利なケースをアドバイスします。

### 電子帳簿保存法対応

2024年1月から義務化された電子取引データの電子保存に対応しています。電子取引（メール添付の請求書、Webダウンロードの領収書等）のデータを取得日時とともに自動保存し、検索要件（日付、金額、取引先名での検索）を満たすインデックスを自動生成します。

JIIMA認証のタイムスタンプサービスとの連携にも対応しており、保存したデータに真正性を付与することが可能です。また、スキャナ保存制度に対応した紙書類のPDFスキャンデータの管理機能も提供しています。

### セキュリティとプライバシー

マイナンバーの取り扱いには最高レベルのセキュリティ対策を適用しています。マイナンバーはAES-256-GCMで暗号化して保存し、メモリ上でも暗号化状態を維持します。復号は電子署名時のみ一時的に行い、処理完了後は即座にメモリから消去します。ログファイルにはマイナンバーの先頭4桁のみがマスク表示され、完全な番号は一切記録されません。

パスワードや証明書のPINコードについても同様に暗号化保存され、平文での保存や表示は行いません。環境変数からの読み取り時にもセキュアな処理を行い、プロセスリストやシェルヒストリーへの漏洩を防止します。

### 確定申告シミュレーション

申告書を作成する前に、税額のシミュレーションを実行できます。収入金額、経費、各種控除の金額を入力すると、所得税・復興特別所得税・住民税・事業税の概算額を即座に計算します。複数のシナリオを比較する機能も搭載しており、例えば「青色申告65万円控除 vs 白色申告」「簡易課税 vs 本則課税」「ふるさと納税の最適額」などのシミュレーションを並列で実行し、最も有利な選択肢を提案します。

シミュレーション結果はJSON形式で保存され、過年度のシミュレーション結果との比較も可能です。税制改正の影響を事前に把握するために、次年度の改正案に基づく仮想シミュレーションも実行できます。

### 税務カレンダーと通知システム

年間の税務スケジュールを一元管理し、各種期限のリマインダーを提供します。Slack、Lark、メール、デスクトップ通知など複数の通知チャネルに対応しており、期限の7日前、3日前、前日、当日の段階的通知をデフォルトで設定しています。

通知内容には期限情報だけでなく、必要な準備事項（例: 消費税確定申告の前に、課税売上割合の確認と仕入税額控除の集計を完了させること）や参考リンクも含まれます。税務署の窓口混雑予測情報と連動し、確定申告期間中は混雑が少ない時間帯の来署を推奨する機能も搭載しています。

### Data Import and Export Capabilities

The E-Tax Agent supports a comprehensive range of data import formats for seamless integration with existing accounting workflows. CSV files from major Japanese accounting platforms are automatically detected and parsed, with column mapping handled intelligently based on header names and data patterns. JSON format is also supported for programmatic data exchange with custom accounting systems or third-party APIs.

Export capabilities include generating standardized XTX files for e-Tax submission, PDF reports for physical archival, and CSV summaries for further analysis in spreadsheet applications. The agent can also generate XML files compatible with the eLTAX system for local tax filing, enabling a unified approach to both national and local tax obligations from a single command-line interface.

For businesses that operate across multiple fiscal periods or manage multiple entities, the agent supports profile-based configuration. Each profile maintains its own set of credentials, tax office codes, and filing history, allowing accountants and tax professionals to efficiently manage multiple clients from a single installation. Profile switching is instantaneous and does not require restarting the agent or reconfiguring environment variables.

Batch processing mode enables the submission of multiple tax returns in sequence, with automatic error handling and retry logic for each submission. This is particularly useful for tax professionals managing dozens or hundreds of client filings during the busy March filing season.
